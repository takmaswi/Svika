### Phase 3 known issues / follow-ups

- **Parcel button is a placeholder** ‚Äî Phase 4 stretch 1 wires the parcel happy path. The button shows "Parcel handover ships in Phase 4" so the conductor screen still feels complete during the demo.
- **No DB transaction on redeem** ‚Äî if `tickets` update succeeds but `vehicles` update fails, ticket is redeemed without the passenger-count bump. Acceptable for the demo. Postgres function wrapper is roadmap.
